# Phase 7: Intelligence & Analytics -- UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (hand-built Tailwind components, consistent with phases 1-6) |
| Preset | not applicable |
| Component library | none (custom Tailwind components) |
| Icon library | none (Unicode symbols and inline SVG) |
| Font | system font stack (Tailwind default: `ui-sans-serif, system-ui, ...`) |

**Rationale:** Project is 75% complete with 6 phases using hand-built Tailwind components. No component library. All UI is utility-class Tailwind. This phase follows the same approach for consistency.

---

## Spacing Scale

Declared values (multiples of 4). Matches existing codebase patterns from PricingTable, ModelDetailClient, TopNav.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge padding |
| sm | 8px | Compact element spacing, filter row gaps |
| md | 16px | Default element spacing, card padding, section padding (px-4) |
| lg | 24px | Section padding between major blocks |
| xl | 32px | Layout gaps, chart container margins |
| 2xl | 48px | Major section breaks (py-12 for empty states) |
| 3xl | 64px | Page-level top/bottom padding (py-16) |

Exceptions: Bell icon touch target at 44px (D-15, accessibility minimum for interactive elements)

## Typography

Existing codebase uses these sizes consistently. This phase declares them formally.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 | Table cells, descriptions, filter labels, card body text |
| Label | 12px (`text-xs`) | 400 (regular) | 1.4 | Badges, timestamps, type labels, meta text (differentiated by size, not weight) |
| Heading | 20px (`text-xl`) | 600 (semibold) | 1.3 | Section headings within pages ("Price Trends", "Active Promotions") |
| Display | 24px (`text-2xl`) | 600 (semibold) | 1.2 | Page-level headings ("Intelligence & Analytics", "Compare Models") |

Font weights in use: 400 (body, labels, badges) and 600 (headings, display, brand/logo).

## Color

Existing codebase uses Tailwind default palette. This phase formalizes the contract.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (white) | Page backgrounds, card surfaces, table backgrounds |
| Secondary (30%) | `#f9fafb` (gray-50) | Table header rows, filter panel backgrounds, empty state backgrounds, expired card backgrounds |
| Accent (10%) | `#2563eb` (blue-600) | Active nav links, model name links, currency toggle active state, "Set Alert" button, "Compare" primary action |
| Destructive | `#dc2626` (red-600) | "Remove alert" action, price increase markers on charts |

**Accent reserved for:**
- Active navigation links in TopNav
- Clickable model name links (navigating to detail pages)
- Active currency toggle state (USD/VND)
- "Set Alert" bell icon when active
- "Compare selected" primary action button
- Price drop markers on trend charts (green-600 `#16a34a` instead of blue)

**Chart line colors (from existing PriceHistoryChart):**
- Input price line: `#3b82f6` (blue-500)
- Output price line: `#ef4444` (red-500)
- Price drop markers: `#16a34a` (green-600)
- Price increase markers: `#dc2626` (red-600)
- New launch markers: `#f59e0b` (amber-500, star icon)

**Badge colors (from existing PromotionsList):**
- Free tier: `bg-green-100 text-green-800`
- Promotion: `bg-blue-100 text-blue-800`
- Beta: `bg-purple-100 text-purple-800`

## Interaction Patterns

### Trend Chart (D-04)

- **Price drop:** Green dot (`#16a34a`, radius 6) at the data point where price decreased
- **Price increase:** Red dot (`#dc2626`, radius 6) at the data point where price increased
- **New model launch:** Amber star icon (`#f59e0b`) at the first data point
- **Hover tooltip:** Shows date, input price, output price, and change percentage
- **Chart height:** 300px (matches existing PriceHistoryChart)
- **Responsive:** `ResponsiveContainer` from Recharts (existing pattern)

### Model Selector (D-11)

- **Type:** Custom searchable dropdown (no external library)
- **Behavior:** Click to open, type to filter, click option to select
- **Display:** Shows model name + provider in dropdown options
- **Max selections:** 5 (disable "Add model" button when 5 selected)
- **URL sync:** Selected models sync to `?models=` query param (D-10)
- **Keyboard:** Arrow keys to navigate, Enter to select, Escape to close

### Alert Banner (D-16)

- **Position:** Fixed, bottom-right corner (`fixed bottom-4 right-4`)
- **Content:** "Price alert: {modelName} is now {currentPrice} (below your {threshold} threshold)"
- **Dismiss:** X button to close individual alert
- **Stack:** Multiple alerts stack vertically
- **Auto-dismiss:** 10 seconds, then fades out
- **Z-index:** z-50 (above page content, below TopNav)

### Bell Icon (D-15)

- **Position:** Next to model name in ModelDetailClient hero section
- **States:**
  - Default: Gray outline bell (`text-gray-400`)
  - Active (alert set): Blue filled bell (`text-blue-600`)
  - Hover: Darker shade of current state
- **Click behavior:** Toggles inline alert form below (not a modal)
- **Inline form:** Shows threshold input + "Set Alert" / "Remove Alert" buttons
- **Touch target:** 44px minimum (padding to reach size)

## Data Flow Notes

### localStorage Schema (D-18)

```typescript
// Key: 'ai-daily-price-alerts'
// Value: JSON array of AlertEntry
interface AlertEntry {
  modelName: string;    // e.g. "gpt-4o"
  sourceId: number;     // extraction source ID
  thresholdPrice: number; // USD per 1M tokens
  createdAt: string;    // ISO date string
}
```

### Alert Check Flow (D-16)

1. Page loads (any page with AlertBanner component)
2. Read `ai-daily-price-alerts` from localStorage
3. For each alert, fetch current price from page data (or cached API)
4. If current price < threshold, show AlertBanner
5. User dismisses or auto-dismiss after 10 seconds

### Compare URL Format (D-10)

```
/compare?models=gpt-4o,claude-sonnet-4-5,gemini-2.0-flash
```

- Comma-separated model names (URL-encoded if needed)
- Page reads `models` param, resolves to extraction data
- 2-5 models enforced client-side
