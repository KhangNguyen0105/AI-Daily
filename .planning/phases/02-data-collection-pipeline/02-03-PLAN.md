**Create src/pipeline/orchestrator.ts:**

The orchestrator coordinates the daily pipeline run across all registered providers.

**Exported types:**
```typescript
export interface PipelineStats {
  totalProviders: number;
  attempted: number;
  succeeded: number;
  failed: number;
  extractions: number;
  verifiedCount: number;
  likelyCount: number;
  lowConfidenceCount: number;
}
```

**Exported functions:**

1. `orchestrateDailyRun(): Promise<number>`
   - Call getAllAdapters() to get all registered providers
   - Create a pipelineRun record in the database:
     ```
     status: 'running'
     startedAt: new Date()
     stats: { totalProviders: adapters.length, attempted: 0, succeeded: 0, failed: 0, extractions: 0, verifiedCount: 0, likelyCount: 0, lowConfidenceCount: 0 }
     ```
   - For each adapter, add a job to collectQueue:
     ```
     collectQueue.add('collect', { providerName: adapter.config.name, pipelineRunId: run.id }, { jobId: `collect-${adapter.config.name}-${Date.now()}` })
     ```
   - Return the pipelineRun.id

2. `updatePipelineStats(pipelineRunId: number, updates: Partial<PipelineStats>): Promise<void>`
   - Fetch the current pipelineRun record
   - Merge updates into existing stats
   - Update the record in the database

3. `finalizePipelineRun(pipelineRunId: number, status: 'completed' | 'failed'): Promise<void>`
   - Update pipelineRun: set status and completedAt

  The extract worker currently imports ScoreJobData from score.ts. After updating score.ts, this will automatically pick up the new type.

**Rewrite src/pipeline/workers/score.ts:**

Replace the Phase 1 pass-through with real verification and scoring. The score worker:

1. **Update ScoreJobData interface:**
   ```typescript
   export interface ScoreJobData {
     extractionIds: number[];
     rawDataId: number;
     sourceId: number;
   }
   ```

2. **In the worker handler:**
   - If extractionIds.length === 0, return early (existing behavior)
   - Fetch raw HTML from rawData table using rawDataId (same pattern as extract worker)
   - Fetch all extraction records from extractions table using extractionIds
   - Look up the source name from sources table using sourceId
   - Get the source tier (default to 'tier1' for now -- all providers crawl official pricing pages)
   - For each extraction:
     a. Call verifyExtraction(html, [extraction], process.env.OPENAI_API_KEY)
     b. Call calculateConfidence(tier, extraction, verificationResult.verified)
     c. Update the extraction record with the new confidence value
     d. If verificationResult.verified === false AND disagreements.length > 0:
        - Log the disagreements as warnings
        - Set confidence to 'low_confidence' (quarantine)
   - Track stats: count verified, likely, low_confidence
   - Chain to generateQueue (preserve existing chaining behavior)

3. **Import additions:**
   ```typescript
   import { verifyExtraction } from '../verification';
   import { calculateConfidence } from '../confidence';
   import { rawData, extractions, sources } from '../../db/schema';
   import { eq, inArray } from 'drizzle-orm';
   ```

4. **Error handling:**
   - If verifyExtraction throws (LLM API error), log the error and fall back to the extraction's existing confidence (don't fail the whole pipeline)
   - Keep the existing error/failed event handlers
