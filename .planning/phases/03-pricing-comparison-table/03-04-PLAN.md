## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External exchange-rate API -> worker | Untrusted public API response enters server-side worker code. |
| Worker -> PostgreSQL | Validated rate is persisted for later public rendering. |
| Worker failure -> daily pipeline | Exchange-rate failure must not stop provider collection. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-01 | Spoofing | Exchange-rate API response | mitigate | Use a constant HTTPS endpoint, validate `result` and finite positive numeric `rates.VND`, and reject malformed payloads. |
| T-03-04-02 | Tampering | Stored exchange rate | mitigate | Store only validated rates; keep fallback chain API -> DB -> `FALLBACK_RATE`. |
| T-03-04-03 | Denial of Service | Daily pipeline worker | mitigate | Bound fetch timeout and catch exchange-rate refresh failures so provider collection still starts. |
| T-03-SC | Tampering | npm installs | accept | No package-manager install is planned. |
</threat_model>
