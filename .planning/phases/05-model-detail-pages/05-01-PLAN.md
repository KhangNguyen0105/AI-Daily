<task type="auto" tdd="true">
  <name>Task 2: Create slug generation and resolution utilities</name>
  <files>app/lib/slug.ts, tests/slug.test.ts</files>
  <behavior>
    - generateSlug("gpt-4o", 1) returns "gpt-4o--1"
    - generateSlug("Claude 3.5 Sonnet", 2) returns "claude-35-sonnet--2"
    - generateSlug("gemini-1.5-pro", 3) returns "gemini-15-pro--3"
    - generateSlug("GPT-4o Mini", 5) returns "gpt-4o-mini--5"
    - generateSlug strips leading/trailing hyphens
    - generateSlug collapses multiple consecutive hyphens into one
    - resolveSlug("gpt-4o--1") returns { modelName: "gpt-4o", sourceId: 1 } (via DB query)
    - resolveSlug("invalid") returns null (no double-dash separator)
    - resolveSlug("model--abc") returns null (sourceId is not a number)
  </behavior>
  <action>
    Create app/lib/slug.ts with two exported functions:

    1. generateSlug(modelName: string, sourceId: number): string
       - Convert modelName to lowercase
       - Replace all non-alphanumeric characters with hyphens: /[^a-z0-9]+/g -> '-'
       - Remove leading/trailing hyphens: /^-+|-+$/g -> ''
       - Append '--' + sourceId
       - Example: ("Claude 3.5 Sonnet", 2) -> "claude-35-sonnet--2"

    2. resolveSlug(slug: string): Promise<{ modelName: string; sourceId: number } | null>
       - Find the last '--' in the slug using lastIndexOf('--')
       - If not found, return null
       - Extract the part after '--' as sourceIdStr
       - parseInt(sourceIdStr, 10) — if NaN, return null
       - Query the database: SELECT DISTINCT modelName, sourceId FROM extractions WHERE sourceId = {parsed sourceId}
       - For each row, check if generateSlug(row.modelName, row.sourceId) === slug
       - Return the matching { modelName, sourceId } or null
       - Use try/catch — if DB is unavailable, return null

    Import db from '@/src/db/index' and extractions from '@/src/db/schema'.
    Import eq from 'drizzle-orm'.

    Create tests/slug.test.ts using Vitest:
    - Test generateSlug with various model names (spaces, dots, mixed case, special chars)
    - Test generateSlug with edge cases (empty string, already-lowercase, Unicode)
    - Note: resolveSlug tests require DB mocking — skip integration tests, test only the parsing logic by extracting it

    For resolveSlug parsing tests, test the slug parsing logic separately:
    - Extract the "parse slug" logic into a helper: parseSlug(slug: string): { sourceId: number } | null
    - Test parseSlug with valid slugs, missing separator, non-numeric sourceId
  </action>
  <verify>
    <automated>pnpm vitest run tests/slug.test.ts</automated>
  </verify>
  <done>generateSlug produces correct deterministic slugs. resolveSlug parses slugs correctly and queries DB. All slug tests pass. Module exports both functions.</done>
