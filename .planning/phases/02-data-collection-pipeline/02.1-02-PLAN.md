<objective>
Build evidence-first verification foundation enabling quote-level source linking and edge-case handling.

**Purpose:** Current verification has no evidence anchoring (cannot trace prices back to source), no edge-case classification (free tiers and special pricing treated as failures), and limited numeric tolerance rules. Phase 2.1 requires quote-level evidence + edge-case handling to prevent false hallucinations and support multi-dimensional confidence scoring (Wave 3).

**Output:** Extended extraction schema with evidence anchoring, evidence capture/retrieval functions, quote-level verification with numeric tolerance rules, edge-case classifier, and enhanced score worker.
</objective>

```typescript
export interface EvidenceAnchor {
  source_url: string;
  extracted_text_snippet: string; // 200-500 char context window
  evidence_quote: string; // exact supporting text
  evidence_selector?: string; // CSS/XPath if available
  evidence_hash: string; // SHA256(evidence_quote)
  extracted_at: Date;
}

export interface EvidenceQuotes {
  model_name?: EvidenceAnchor;
  input_price?: EvidenceAnchor;
  output_price?: EvidenceAnchor;
  context_window?: EvidenceAnchor;
  currency?: EvidenceAnchor;
  pricing_unit?: EvidenceAnchor;
  free_tier?: EvidenceAnchor;
  effective_date?: EvidenceAnchor;
}

export function captureEvidence(
  html: string,
  extractedField: string, // 'model_name', 'input_price', etc.
  fieldValue: string | number,
  sourceUrl: string,
): EvidenceAnchor

export function generateEvidenceSelector(
  html: string,
  targetText: string,
): string | undefined // CSS selector if found

export function validateEvidenceQuote(
  quote: string,
  fieldValue: string | number,
): { valid: boolean; confidence: 'high' | 'medium' | 'low' }
```

**captureEvidence logic:**
- Search HTML for fieldValue or close matches (exact, case-insensitive, with punctuation variations)
- Extract 200-500 char context window around found text
- Store exact quote as evidence_quote
- Use generateEvidenceSelector to find CSS selector or XPath
- Compute SHA256 hash of quote
- If value not found in HTML: return null (field cannot be verified)

**Per D-08:** No price is verified unless anchored to specific source snippet. If extraction cannot find supporting quote in HTML → set verification_status = needs_review.

  tolerance?: number,

export function detectLargeChange(
  pass1Result: ExtractionResult,
  previousExtraction: ExtractionResult | null,
): { changed: boolean; changeType: 'price_increase' | 'price_decrease' | 'unit_change' | 'field_disappear'; percentChange: number }

export async function verifyWithEvidenceQuotes(
  pass1: ExtractionResult,
  evidenceQuotes: EvidenceQuotes,
  htmlSnapshot: string,
): Promise<{ verified: boolean; missingEvidence: string[]; quoteMismatches: string[] }>
```

**Numeric tolerance rules (per D-08):**
- Same unit (e.g., $0.003/1K vs $3/1M if arithmetic is identical): ±0.1% tolerance
- Converted unit (e.g., $0.003/1K user enters as $0.000003/1 token): ±0.5% tolerance (accounts for FX, rounding)
- Currency conversion: depends on FX source/date (store FX rate used)
- Tiered/plan pricing: do NOT compare as flat token pricing (mark as edge_case, not comparable)

**Large change detection (per D-08):**
- If price changed >20% compared to previous extraction: set price_change_flag=true
- Trigger stronger verification: evidence quote required, manual review recommended
- Compare pass1 result against pass2 with lower tolerance
- Examples:
  - Input $1 → $1.50: 50% increase → flag + escalate
  - Input $1 → $2: price doubled → flag + escalate
  - Field missing that was previously present: flag + escalate
  - Unit changed (per 1K → per 1M without recalculation): flag + escalate

**Per D-08:** Large change examples: >20% price diff, input/output swapped, missing output price, model disappeared, pricing unit changed → require stronger verification.

**Hallucination prevention (per D-08):**
- Missing price → null NOT guessed
- "Contact sales" → pricing_model_type = enterprise_only NOT invented price
- No context window found → null with confidence tracking
- Ambiguous model family → needs_review NOT assumed
- No evidence quote → CANNOT be verified; mark needs_review or low_confidence

export interface EdgeCaseFlags {
  free_tier?: { detected: boolean; text: string; evidence_quote?: string };
  free_trial_credits?: { detected: boolean; amount: number; currency: string };
  cached_input_pricing?: { detected: boolean; rate: string };
  batch_discounts?: { detected: boolean; discount_pct: number };
  reasoning_tokens?: { detected: boolean; rate_per_1m: string };
  image_audio_video_pricing?: { detected: boolean; unit: string };
  per_request_pricing?: { detected: boolean; rate: string };
  monthly_plans?: { detected: boolean; price: string; includes_tokens?: string };
  enterprise_only?: { detected: boolean; contact_sales: boolean };
  contact_sales_pricing?: { detected: boolean };
  region_specific_pricing?: { detected: boolean; regions: string[] };
  deprecated_model?: { detected: boolean; deprecation_date?: string };
  promotional_pricing?: { detected: boolean; promo_text: string };
  fine_tuning_pricing?: { detected: boolean; rate: string };
  embedding_pricing?: { detected: boolean; rate: string };
}

export async function classifyEdgeCases(
  htmlSnapshot: string,
  extraction: ExtractionResult,
  providerName: string,
): Promise<EdgeCaseFlags>

export function isComparableTokenPricing(flags: EdgeCaseFlags): boolean
```

**Detection logic (per D-08):**
- Scan HTML for keywords: "free", "trial", "cache", "batch", "discount", "reasoning", "image", "audio", "video", "request", "month", "enterprise", "contact sales", "region", "deprecated", "promo", "fine-tuning", "embedding"
- For each detected category: extract relevant text as evidence
- `isComparableTokenPricing()`: returns false if free_tier OR monthly_plans OR enterprise_only OR contact_sales_pricing is detected
  - Used to filter Cost Calculator (only show comparable token-based pricing)
  - Used in ranking (exclude non-comparable from "best price" rankings)

**Update extract worker (src/pipeline/workers/extract.ts):**

After calling adapter.extract():
1. Call classifyEdgeCases(html, extractionResult, providerName) → get edgeCaseFlags
2. For each field in extraction, call captureEvidence() → build evidenceQuotes JSONB
3. Check if core fields have evidence:
   - If model_name, input_price, or output_price missing evidence: set verification_status = needs_review
4. Detect large price changes via detectLargeChange() against previous extraction (if exists)
   - If true: set price_change_flag = true
5. Pass to score worker:
   ```
   await scoreQueue.add({
     extractionId: ...,
     evidenceQuotes: {...},
     edgeCaseFlags: {...},
     priceChangeFlag: detected,
     pass1Values: extractionResult
   })
   ```

**Update score worker (src/pipeline/workers/score.ts):**

In verification pass:
1. Receive evidenceQuotes, edgeCaseFlags, priceChangeFlag from extract worker
2. Call verifyWithEvidenceQuotes(pass1, evidenceQuotes, htmlSnapshot) → check quotes match pass2
3. If large price change: apply lower tolerance (0.05% instead of 0.1%) for pass1-vs-pass2 comparison
4. If edge case detected: set verification_status = verified_with_warning (not just verified)
5. Store all edge case flags in extractions.edge_case_flags JSONB
6. For quarantine logic:
   - If verification disagreement AND NOT edge_case: quarantine (status = conflicted)
   - If verification disagreement AND edge_case (e.g., free tier state changed): status = verified_with_warning + add note
7. Update extraction with verification_status, verification_notes, edge_case_flags

**Backward compatibility:**
- Existing extractions without evidence_quotes: treat as needs_review (safer default)
- Existing extractions without edge_case_flags: assume no edge cases
- Extract worker calls classifyEdgeCases for all new extractions

Per D-08: Edge cases are explicitly classified and not forced into normal fields. Verification must handle them separately.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTML Snapshot → Evidence Quote | Untrusted HTML; must validate evidence quote exists in snapshot before storing |
| Evidence Quote → Verification | Evidence quote is source of truth; pass2 verification MUST compare against evidence, not pass1 values |
| LLM Extraction → Evidence Anchor | LLM may hallucinate prices not in source; evidence anchoring proves or disproves |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Tampering | Evidence quote modification | mitigate | evidence_hash (SHA256) stored with quote; changes detected via hash mismatch during audit |
| T-02-07 | Repudiation | Evidence selector spoofing | mitigate | CSS selector validated against actual HTML in verification pass; if selector no longer finds element → needs_review |
| T-02-08 | Information Disclosure | Historical evidence snapshots | mitigate | raw_html_snapshot stored separately (Phase 2.2); evidence quotes are immutable once recorded |
| T-02-09 | Denial of Service | Evidence search performance | mitigate | captureEvidence uses indexOf + substring (O(n) linear); bounded to first 100KB of HTML; time-limited to 5 sec |
| T-02-10 | Spoofing | LLM hallucination of prices | mitigate | If price not found in evidence quote → verification_status = needs_review; cannot be marked verified without evidence anchor |

</threat_model>
