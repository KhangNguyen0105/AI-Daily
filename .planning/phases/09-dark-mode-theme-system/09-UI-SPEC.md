## Color — Dual-Theme Contract

Phase 9 introduces CSS custom properties in `globals.css`. All hardcoded Tailwind color classes (`bg-white`, `text-gray-900`, `border-gray-200`, etc.) in 33 component files (273 occurrences) must be replaced with token references.

### CSS Custom Properties (defined in `globals.css`)

**Naming convention:** `--{category}-{semantic}`

```css

  --bg-secondary: #f9fafb;       /* 30% — cards, sidebar, nav, table rows */
  --bg-tertiary: #f3f4f6;        /* Hover states, active row highlight */

  --text-primary: #111827;        /* Headings, primary content */
  --text-secondary: #6b7280;      /* Labels, descriptions, muted text */
  --text-tertiary: #9ca3af;       /* Placeholder, disabled */

  /* Borders */
  --border-primary: #e5e7eb;      /* Default borders */
  --border-secondary: #d1d5db;    /* Emphasized borders */

  /* Accent (unchanged across themes per D-04) */
  --accent-blue: #2563eb;         /* Links, active nav, primary CTA */
  --accent-green: #22c55e;        /* Positive indicators, free tier badge */
  --accent-yellow: #eab308;       /* Warnings, low-confidence badge */
  --accent-red: #ef4444;          /* Errors, destructive actions, price drops */
```

### Tailwind Integration

Use Tailwind CSS 4 `@theme` directive to register tokens as Tailwind utilities:

```css
@theme {
  --color-bg-primary: var(--bg-primary);
  --color-bg-secondary: var(--bg-secondary);
  --color-bg-tertiary: var(--bg-tertiary);
  --color-text-primary: var(--text-primary);
  --color-text-secondary: var(--text-secondary);
  --color-text-tertiary: var(--text-tertiary);
  --color-border-primary: var(--border-primary);
  --color-border-secondary: var(--border-secondary);
}
```

This enables usage as `bg-bg-primary`, `text-text-primary`, `border-border-primary` in JSX.

## ThemeToggle Component Contract

**Location:** `app/components/ThemeToggle.tsx` (new file)
**Placement:** Inside `TopNav.tsx`, after the nav links, before closing `</div>`

```
+--[ AI Daily ]--[ Pricing ]--[ Digest ]--[ Trends ]--[ ... ]--[ (sun/moon) ]--+
```

**Behavior:**
1. Renders a `<button>` with icon-only appearance (no text label)
2. Reads current theme from `ThemeContext` (React context)
3. On click: toggles `dark` class on `<html>`, writes to `localStorage('theme')`, updates context
4. Icon animates: `transition: transform 200ms` on toggle (rotate 180deg)
5. Button size: 36x36px (icon-only touch target, exception to 8pt grid — per accessibility minimum)
6. Button styling: `rounded-lg bg-bg-tertiary hover:bg-border-primary text-text-primary`

## ThemeProvider Component Contract

**Location:** `app/components/ThemeProvider.tsx` (new file)
**Wraps:** Children in `app/layout.tsx`

**Behavior:**
1. React context providing `{ theme: 'light' | 'dark', toggle: () => void }`
2. Initializes from `document.documentElement.classList.contains('dark')` after hydration
3. `toggle()` function: adds/removes `.dark` class on `<html>`, writes to `localStorage`
4. Does NOT set initial theme — that is handled by inline script (SSR-safe per D-06)

---

## Inline Script Contract (SSR Flash Prevention)

**Location:** `<head>` of `app/layout.tsx` (per D-06)

**Script behavior:**
1. Reads `localStorage.getItem('theme')`
2. If value is `'dark'` OR (value is null AND `window.matchMedia('(prefers-color-scheme: dark)').matches`): add `class="dark"` to `<html>`
3. Otherwise: ensure no `.dark` class
4. Must execute BEFORE React hydration to prevent flash

**Next.js integration:** Add `suppressHydrationWarning` to `<html>` tag in layout.tsx to suppress React hydration mismatch warning caused by the inline script modifying the class.
