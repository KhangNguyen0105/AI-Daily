**Create src/pipeline/scheduler.ts:**

The scheduler sets up a BullMQ repeatable job that triggers the daily pipeline.

**Exported functions:**

1. `setupDailyScheduler(): Promise<void>`
   - Create a new Queue instance named 'daily-pipeline' using the shared redisConnection
   - Call queue.add() with:
     ```
     name: 'daily-collection'
     data: {}
     opts: {
       repeat: {
         pattern: '0 6 * * *',  // Daily at 6:00 AM UTC
         key: 'daily-collection',  // Prevents duplicates on restart
       },
       removeOnComplete: { count: 5 },
       removeOnFail: { count: 10 },
     }
     ```
   - Log: 'Daily scheduler configured: 0 6 * * * (6 AM UTC)'

2. `removeDailyScheduler(): Promise<void>`
   - Create a Queue instance named 'daily-pipeline'
   - Call queue.removeRepeatable('daily-collection', { pattern: '0 6 * * *', key: 'daily-collection' })
   - Close the queue

3. `createDailyPipelineWorker(): Worker`
   - Create a Worker for the 'daily-pipeline' queue
   - The worker handler calls orchestrateDailyRun() (from orchestrator.ts)
   - The worker handler is the entry point that triggers the full pipeline
   - Return the worker instance
   - Include error/completed/failed event handlers (same pattern as other workers)

**Note on architecture:** The 'daily-pipeline' queue is a separate queue from the 4 pipeline stage queues. Its sole purpose is to trigger orchestrateDailyRun() on a cron schedule. The orchestrator then enqueues collect jobs into the existing collectQueue.
