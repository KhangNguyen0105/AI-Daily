## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PostgreSQL -> Server Component | Pipeline-collected pricing data crosses into the public page render. |
| Server Component -> Client Component | Pricing rows and freshness metadata are serialized into client props. |
| Exchange-rate worker -> Server Component | Latest exchange rate may be unavailable or stale, but must not block pricing display. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Tampering | `app/page.tsx` pricing query | mitigate | Deduplicate in SQL to latest `collectedAt` per model+sourceId, preserving source joins and avoiding stale historical rows in the current table. |
| T-03-02-02 | Denial of Service | DB or exchange-rate failure | mitigate | Keep independent try/catch blocks; DB failure produces loadError plus empty data, and build/ISR exchange-rate refresh failure falls back to stored DB rate then `FALLBACK_RATE`. |
| T-03-02-03 | Repudiation | Data freshness metadata | mitigate | Display last collection time and next scheduled run time so users can identify stale pipeline output. |
| T-03-SC | Tampering | npm installs | accept | No package-manager install is planned. |
</threat_model>
