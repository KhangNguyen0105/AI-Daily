# Phase 6 — UI Design Contract

> Visual and interaction contract for the Daily Content Engine frontend surfaces. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (Tailwind utility classes only) |
| Icon library | none (HTML entities for arrows: `&larr;`) |
| Font | system default (`font-sans` — Tailwind default) |

**Rationale:** No `components.json` or Tailwind config exists. The app uses Tailwind v4 with `@import "tailwindcss"` and no custom `@theme` tokens. All styling is done via Tailwind utility classes with default values. This is consistent with the existing codebase — no design system tool is initialized.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`, `p-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `mb-2`, `py-1`, `px-2`) |
| md | 16px | Default element spacing (`gap-4`, `mb-4`, `px-4`, `py-4`) |
| lg | 24px | Section padding (`mb-6`) |
| xl | 32px | Layout gaps (`py-8`) |
| 2xl | 48px | Major section breaks (`pb-16`) |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions: TopNav height uses `py-3` (12px) + `h-14` (56px total) for compact fixed header.

**Source:** Derived from existing component patterns in `HomePageClient.tsx` (`mb-6`, `pb-16`), `ModelDetailClient.tsx` (`py-8`, `mb-4`, `mb-8`), and `page.tsx` (`py-8`, `px-4`).

## Surface Specifications

### Surface 1: TopNav Component

**File:** `app/components/TopNav.tsx`
**Type:** Client component (`'use client'` — needs `usePathname()` for active state)
**Analog:** None (new surface — no existing nav component)

**Layout:**
```
+------------------------------------------------------------------+
| AI Daily          Pricing    Daily Digest                         |
+------------------------------------------------------------------+
```

**Structure:**
- Fixed top, full width: `fixed top-0 left-0 right-0 z-50`
- Background: `bg-white border-b border-gray-200`
- Inner container: `max-w-6xl mx-auto px-4 py-3 flex items-center justify-between`
- Brand: `<Link href="/">` with `text-lg font-bold text-gray-900`
- Nav links: `flex items-center gap-6`
- Each link: `<Link>` with conditional classes:
  - Active: `text-sm font-semibold text-blue-600`
  - Inactive: `text-sm text-gray-600 hover:text-gray-900`

**Active detection:** `usePathname()` — link is "active" when pathname starts with the link's href (e.g., `/digest` is active for both `/digest` and `/digest/2026-06-14`).

**Height:** `py-3` (12px top + 12px bottom) + content ~20px = ~44px. Use `h-14` (56px) as fixed height for consistent `pt-14` offset on body content.

**Responsive:** No breakpoints needed — two links fit comfortably on mobile. If future links overflow, add `hidden md:flex` for desktop-only items.

### Surface 2: Layout Update

**File:** `app/layout.tsx`
**Type:** Server component (existing)
**Change:** Import TopNav, add before `{children}`, add top padding.

**Updated structure:**
```tsx
<html lang="en">
  <body>
    <TopNav />
    <div className="pt-14">{children}</div>
  </body>
</html>
```

**Impact:** All existing pages (home, model detail) will now have the TopNav. The `pt-14` ensures content is not hidden under the fixed nav. Existing pages already have `py-8` spacing, so the additional 56px top padding is additive and non-breaking.

### Surface 3: /digest Archive Page

**File:** `app/digest/page.tsx`
**Type:** Server component (no `'use client'`)
**Analog:** `app/page.tsx` (server component with Drizzle query + ISR)

**Layout:**
```
+------------------------------------------------------------------+
|                                                                     |
|                    Daily Digest Archive                              |
|                                                                     |
|   June 14, 2026                                                     |
|   GPT-4o price drops 20%, new Gemini model launched               |
|   Summary text here...                                              |
|                                                                     |
|   June 13, 2026                                                     |
|   No pricing changes detected today. 15 models tracked.            |
|   Summary text here...                                              |
|                                                                     |
|   Load more                                                         |
|                                                                     |
+------------------------------------------------------------------+
```

**Structure:**
- Container: `max-w-4xl mx-auto px-4 py-8`
- Page title: `<h1 className="text-3xl font-bold tracking-tight mb-8">Daily Digest Archive</h1>`
- Article list: `<div className="space-y-0">` (no gap — use border separators)
- Each article item: `<article>` with `border-b border-gray-200 py-6` (last item: `border-b-0` or use `last:`)
  - Date: `<time className="text-sm text-gray-500">{format(date, 'MMMM d, yyyy')}</time>`
  - Title: `<Link href={/digest/${date}} className="text-lg font-semibold text-gray-900 hover:text-blue-600 mt-1 block">`
  - Summary: `<p className="text-sm text-gray-600 mt-1">{summary}</p>`
- Load more: `<Link href={/digest?offset=${offset}} className="text-blue-600 hover:text-blue-800 text-sm mt-8 inline-block">Load more</Link>`
- Empty state: `<div className="text-center py-16"><p className="text-gray-500">No articles yet. Check back after the first pipeline run.</p></div>`

**Responsive:** Single column layout works on all screen sizes. No breakpoints needed.

**Pagination:** Server-side via `searchParams.offset`. Query `.offset(offset).limit(30)`. If results < 30, hide "Load more". Parse offset as integer, default to 0.

### Surface 5: DigestArticle Component

**File:** `app/components/DigestArticle.tsx`
**Type:** Client component (`'use client'` — react-markdown requires client-side rendering)
**Analog:** None (new surface)

**Structure:**
```tsx
<div className="max-w-4xl mx-auto px-4 py-8">
  <Link href="/digest" className="text-blue-600 hover:text-blue-800 text-sm mb-4 inline-flex items-center gap-1">
    &larr; Back to digest
  </Link>

  <h1 className="text-3xl font-bold tracking-tight">{article.title}</h1>

  <p className="text-sm text-gray-500 mt-2">
    Published: {format(new Date(article.publishedAt || article.date), 'MMMM d, yyyy')}
  </p>

  <div className="mt-8 prose prose-slate max-w-none prose-headings:font-semibold prose-h2:text-xl prose-h2:mt-8 prose-h2:mb-4 prose-p:text-gray-700 prose-p:leading-relaxed prose-li:text-gray-700 prose-strong:text-gray-900">
    <ReactMarkdown remarkPlugins={[remarkGfm]}>
      {article.content}
    </ReactMarkdown>
  </div>
</div>
```

**Markdown prose styling (via Tailwind Typography plugin classes or manual overrides):**
- `prose` — base typography
- `prose-slate` — slate color scheme (gray-700 body, gray-900 headings)
- `prose-headings:font-semibold` — all headings semibold
- `prose-h2:text-xl` — h2 at 20px
- `prose-h2:mt-8 prose-h2:mb-4` — h2 spacing (32px top, 16px bottom)
- `prose-p:text-gray-700 prose-p:leading-relaxed` — body text gray-700, 1.625 line-height
- `prose-li:text-gray-700` — list items match body
- `prose-strong:text-gray-900` — bold text dark
- `max-w-none` — remove prose max-width constraint (container handles it)

**Note:** The `prose` classes require `@tailwindcss/typography` plugin. If not installed, the executor must run `pnpm add @tailwindcss/typography` and add `@plugin "@tailwindcss/typography"` to `globals.css`. Alternatively, style markdown elements manually with Tailwind classes via react-markdown's `components` prop.

**Fallback if typography plugin is not desired:** Use react-markdown's `components` prop to style each element:
```tsx
<ReactMarkdown
  remarkPlugins={[remarkGfm]}
  components={{
    h1: ({children}) => <h1 className="text-3xl font-bold mt-8 mb-4">{children}</h1>,
    h2: ({children}) => <h2 className="text-xl font-semibold mt-8 mb-4">{children}</h2>,
    p: ({children}) => <p className="text-gray-700 leading-relaxed mb-4">{children}</p>,
    ul: ({children}) => <ul className="list-disc pl-6 mb-4 text-gray-700 space-y-1">{children}</ul>,
    ol: ({children}) => <ol className="list-decimal pl-6 mb-4 text-gray-700 space-y-1">{children}</ol>,
    li: ({children}) => <li className="text-gray-700">{children}</li>,
    strong: ({children}) => <strong className="font-semibold text-gray-900">{children}</strong>,
    a: ({href, children}) => <a href={href} className="text-blue-600 hover:text-blue-800 underline">{children}</a>,
    blockquote: ({children}) => <blockquote className="border-l-4 border-gray-200 pl-4 italic text-gray-600 my-4">{children}</blockquote>,
    code: ({children}) => <code className="bg-gray-100 px-1.5 py-0.5 rounded text-sm font-mono">{children}</code>,
    pre: ({children}) => <pre className="bg-gray-100 p-4 rounded-lg overflow-x-auto my-4">{children}</pre>,
    table: ({children}) => <table className="min-w-full border-collapse border border-gray-200 my-4">{children}</table>,
    th: ({children}) => <th className="border border-gray-200 px-4 py-2 bg-gray-50 text-left font-semibold">{children}</th>,
    td: ({children}) => <td className="border border-gray-200 px-4 py-2">{children}</td>,
  }}
>
  {article.content}
</ReactMarkdown>
```

**Decision:** Use the `components` prop approach (no additional plugin dependency). This is explicit, Tailwind-native, and matches the project's existing pattern of no design system tools.

## Interaction States

### TopNav Links
| State | Style |
|-------|-------|
| Default (inactive) | `text-sm text-gray-600` |
| Hover (inactive) | `text-sm text-gray-600 hover:text-gray-900` |
| Active (current page) | `text-sm font-semibold text-blue-600` |
| Focus | Browser default outline (no custom style) |

### Archive Article Links
| State | Style |
|-------|-------|
| Default | `text-lg font-semibold text-gray-900` |
| Hover | `text-lg font-semibold text-gray-900 hover:text-blue-600` |
| Focus | Browser default outline |

### "Back to digest" / "Back to pricing" Links
| State | Style |
|-------|-------|
| Default | `text-blue-600 hover:text-blue-800 text-sm inline-flex items-center gap-1` |
| Focus | Browser default outline |

### "Load more" Link
| State | Style |
|-------|-------|
| Default | `text-blue-600 text-sm mt-8 inline-block` |
| Hover | `text-blue-600 hover:text-blue-800 text-sm mt-8 inline-block` |
| Focus | Browser default outline |

### Loading States
- **ISR pages:** Next.js handles loading — no custom loading UI needed. Pages are pre-rendered at build time and revalidated every 60 seconds.
- **First visit before ISR:** `generateStaticParams` returns empty array if DB unavailable. Pages will 404 until first ISR revalidation catches up. This is the established pattern from `/model/[slug]`.

### Empty States
- **Archive page (no articles):** Centered message: `No articles yet. Check back after the first pipeline run.` with `text-gray-500 text-center py-16`.
- **Article page (invalid date):** `notFound()` — Next.js default 404 page.
- **Article page (no article for date):** `notFound()` — Next.js default 404 page.

### Error States
- **DB unavailable:** Graceful fallback to empty array. Archive shows empty state. Article page calls `notFound()`.
- **No custom error boundaries:** Follow existing pattern — `try/catch` with fallback, no `error.tsx` files.

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Keyboard navigation | All links are native `<Link>` elements — keyboard accessible by default. Tab order follows DOM order. |
| ARIA labels | TopNav: `<nav aria-label="Main navigation">`. Archive list: `<main>` landmark. Article: `<article>` semantic element. |
| Color contrast | All text/background combinations meet WCAG AA (4.5:1 for body text, 3:1 for large text). Gray-500 on white = 4.6:1. Gray-900 on white = 15.4:1. Blue-600 on white = 4.6:1. |
| Screen readers | Article title announced as heading. Published date announced as text. Links have descriptive text (no "click here"). |
| Focus indicators | Browser default — visible focus ring on all interactive elements. |
