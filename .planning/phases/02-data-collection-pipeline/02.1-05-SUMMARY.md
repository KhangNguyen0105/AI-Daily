# Phase 2.1 Plan 05: Tier 2+ Providers and Multi-Source Discovery Summary

Tier 2 provider coverage (7 providers) with multi-source discovery pipeline and model lifecycle tracking.

## What Was Built

### Tier 2 Provider Adapters (4 new, 3 reclassified)
- **OpenRouter** — routing layer aggregating multiple providers; API model discovery via /api/v1/models
- **Nebius** — GPU/compute and LLM endpoints; region-specific pricing
- **SambaNova** — API-first LLM provider; per-token pricing
- **Lepton AI** — per 1M tokens + serverless compute
- **Together, Fireworks, Cohere** — reclassified from Tier 1 to Tier 2

Each adapter implements: crawl (via base class PlaywrightCrawler), extract (via generateObject with pricingSchema), normalize (raw values preserved), evidence anchoring (D-08).

### Provider Registry Updates
- TIER2_PROVIDERS: 7 providers (openrouter, nebius, sambanova, lepton, together, fireworks, cohere)
- TIER3_PROVIDERS: 1 provider (bedrock) — aggregator, lower priority
- getAllTier2Adapters(), getAllTier3Adapters(), getAllProviders() functions
- isTier2Provider(), isTier3Provider(), getProviderTier() helpers
- 18 total providers registered (10 Tier 1 + 7 Tier 2 + 1 Tier 3)

### Model Discovery Schema (src/db/schema.ts)
- **discoveredModels** table — tracks models detected before pricing is available
  - Status lifecycle: announced → pricing_pending → verified → deprecated → replaced → quarantined
  - Source types: pricing_page, feed, api, diff
  - Evidence fields: evidenceUrl, evidenceText (T-02-19 admin verification)
  - Timing: firstDiscoveredAt, pricingFoundAt, deprecatedAt
- **modelStatusEvents** table — immutable audit trail of lifecycle changes
  - Event types: announced, pricing_detected, deprecated, replaced, quarantined
  - triggeredBy tracking: feed_monitor, api_discovery, price_crawler, diff_detection

### Feed Monitoring Worker (src/pipeline/feed-monitor-worker.ts)
- **processChangelog()** — fetches changelog/RSS feeds with 10-second timeout (T-02-22)
- **detectNewModels()** — pattern-based model name extraction from feed text
- **queryProviderAPI()** — queries /v1/models, /api/v1/models, /models endpoints
- **compareModelSnapshots()** — diff-based detection of new/renamed/deprecated models and >20% price changes
- **monitorProviderFeeds()** — main entry point, iterates all providers through all discovery sources
- **createFeedMonitorWorker()** — BullMQ worker for feed-monitor queue
- **recordDiscoveredModel()** — deduplicating insert with status progression

### Scheduler Updates (src/pipeline/scheduler.ts)
- Tier 2 refresh: `0 0,12 * * *` (every 12 hours at 00:00, 12:00 UTC)
- Feed monitoring: `0 2 * * *` (2 AM UTC daily, before main orchestrator)
- createTier2RefreshWorker() — enqueues Tier 2 collect jobs with priority=5
- initializeAllSchedulers() — startup function for all 4 schedulers
- createAllWorkers() — returns all 4 workers for graceful shutdown

### Orchestrator Updates (src/pipeline/orchestrator.ts)
- Per-tier stats: tier2_succeeded, tier2_failed, tier3_succeeded, tier3_failed
- Per-tier provider counts: tier2ProvidersCount, tier3ProvidersCount
- Priority assignment: Tier 1=3, Tier 2=2, Tier 3=1 (BullMQ lower=higher priority)

## Decisions Applied
- **D-01**: 30+ provider coverage with tiering (10 Tier 1 + 7 Tier 2 + 1 Tier 3)
- **D-02**: Multi-source discovery (pricing pages + feeds + APIs + diffs)
- **D-03**: Tier 2 crawl every 12 hours; Tier 3 daily
- **D-04**: Model lifecycle tracking (announced → deprecated)
- **D-08**: Evidence anchoring for Tier 2 adapters

## Threat Mitigations
- **T-02-19**: Evidence URL + text stored in discovered_models for admin verification
- **T-02-20**: Rename detection requires context_window + family match
- **T-02-22**: Feed request 10-second timeout; failures don't block main orchestrator

## Known Stubs
- Tier 3 providers: Only Bedrock registered; Azure OpenAI, Vertex AI, Baseten, OctoAI, Anyscale listed in plan but not implemented (future expansion)
- Feed changelog URLs: Most providers don't expose changelog URLs in config.sources; API discovery is the primary Tier 2 discovery mechanism
