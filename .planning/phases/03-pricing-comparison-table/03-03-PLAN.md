## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> Client filter state | Search and numeric filters are untrusted browser input but remain client-side only. |
| Provider/source data -> DOM | Provider names, model names, confidence, and URLs originate from collected data. |
| Client currency state -> Price display | Currency state changes display formatting only; DB prices remain USD source of truth. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Tampering | Search/filter controls | accept | Controls are client-side only and never sent to the server in this phase; React escapes visible text. |
| T-03-03-02 | Spoofing | Source links | mitigate | Validate URL protocol as `http:` or `https:` before rendering anchors; otherwise render non-link fallback. |
| T-03-03-03 | Tampering | Model/provider display | mitigate | Use `sanitizeDisplayName()` for visible model and provider strings before rendering. |
| T-03-03-04 | Information Disclosure | External source navigation | mitigate | Render public source URLs as provider-name links with URL `href` and `title`, `target="_blank"`, and `rel="noopener noreferrer"`. |
| T-03-SC | Tampering | npm installs | accept | No package-manager install is planned; use existing `@tanstack/react-table` and local Tailwind components. |
</threat_model>
