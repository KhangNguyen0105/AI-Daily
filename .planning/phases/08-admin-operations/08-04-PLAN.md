Build the pipeline monitoring and source management sections: pipeline runs table with expandable details, error log, re-crawl and regenerate triggers, auto-publish toggle, sources table with trust toggle and filters. Also create all supporting API routes.

Purpose: These are the operational controls that let the admin manage data quality and pipeline health. The triggers and toggles give the admin direct control over the automated pipeline.
