## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Sort indicator gap, confidence badge vertical padding |
| sm | 8px | Provider logo/text gap, small control gaps, compact card spacing, compact control and table row vertical padding (`py-2`) |
| md | 16px | Table cell horizontal padding (`px-4`), default panel padding, form control horizontal rhythm |
| lg | 24px | Gap between PricingTable and CostCalculator (`gap-6`), compact page header vertical padding |
| xl | 32px | Page bottom padding and major local grouping when needed |
| 2xl | 48px | Empty/loading state vertical padding (`py-12`) |
| 3xl | 64px | Reserved for page-level separation only; avoid inside the table surface |

Exceptions:
- Text buttons and form controls use 8px vertical padding (`py-2`) where compact density is needed.
- Mobile interactive controls must maintain a practical hit target of at least 40px high; use 44px only for icon-only controls, which Phase 3 does not introduce.
- Table row density stays compact but readable: use `py-2` for cell vertical padding and do not introduce non-standard vertical spacing values.

Source: preserved from existing `PricingTable`, `HomePageClient`, and user decisions D-01 through D-10. Corrected the baseline token mapping so `gap-6` is documented as 24px, not 32px.

## Typography

Use exactly four type sizes and two font weights for Phase 3 surfaces.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 12px (`text-xs`) | 400 regular or 600 semibold | 1.4 |
| Body | 14px (`text-sm`) | 400 regular | 1.5 |
| Heading | 20px (`text-xl`) | 600 semibold | 1.2 |
| Display | 28px (`text-2xl`) | 600 semibold | 1.2 |

Weight contract:
- Allowed weights: 400 and 600 only.
- Normalize existing 500 and 700 utility usage to 600 or 400 when Phase 3 UI is touched.
- Do not add extra weights for badges, rank numbers, nav, or buttons.

Usage:
- Table cells, filters, pagination, row counts, last-updated text: 14px.
- Field legends, badges, metadata, token count details: 12px.
- Section headings: 20px semibold.
- Page title: 28px semibold with compact spacing.

Source: baseline UI-SPEC, workflow typography limit, and existing Phase 3 components. Updated to satisfy the two-weight contract while preserving the same visual hierarchy.

## Interaction Contract

### Table Sorting

- Trigger: click any sortable column header.
- Cycle: unsorted -> ascending -> descending -> unsorted.
- Visual state: unsorted shows a muted bidirectional indicator; sorted shows ascending or descending indicator in `gray-700`.
- Sorting is single-column from the user's perspective.
- Source column is not sortable.
- Header style: `cursor-pointer select-none hover:bg-gray-100 transition-colors`.

### Global Search

- Trigger: controlled text input.
- Placeholder: `Search models, providers, or families...`.
- Performance: use React `useDeferredValue`; do not add debounce dependencies.
- Scope: model name, provider name, and derived model family.
- Reset: `Clear filters` clears the search input.
- Accessibility: visible input plus `aria-label="Search models, providers, or families"`.

### Provider Filter

- Trigger: native select.
- Options: `All Providers` plus sorted unique provider names derived from current data.
- Behavior: exact provider match.
- Accessibility: `aria-label="Filter by provider"`.

### Free-Tier Filter

- Trigger: checkbox.
- Visible label: `Free tier only`.
- Logic: row is free-tier when `inputPricePer1m === 0 && outputPricePer1m === 0`.
- Accessibility: checkbox must be associated with the visible label.

### Advanced Filters

- Trigger: `Advanced Filters` disclosure button.
- State: expose `aria-expanded`.
- Indicator: text indicator only; no icon library.
- Content: three fieldsets: input price range, output price range, context window range.
- Inputs: numeric min/max controls with explicit aria labels.
- Layout: stacked on mobile, row layout from `md` upward.
- Null numeric values pass through range filters so unknown prices are not hidden by default.

### Currency Toggle

- Trigger: two-button group, USD and VND.
- Accessibility: `role="group" aria-label="Toggle currency"`.
- Active state: `bg-blue-600 text-white border-blue-600`.
- Inactive state: `bg-white text-gray-700 border-gray-300 hover:bg-gray-50`.
- Effect: all pricing columns and CostCalculator amounts update in place without losing filters, sorting, or pagination.
- Exchange-rate failure: keep pricing visible using `FALLBACK_RATE`; do not block the table.

### Clear Filters

- Visibility: only when any filter/search/range value is active.
- Action: resets search, provider, free-tier, input/output price range, and context-window range.
- Accessibility: `aria-label="Clear all filters"`.

### Source Attribution

- Source link text: provider name.
- Target: `_blank`.
- Rel: `noopener noreferrer`.
- Invalid or missing URL: show provider name or `Source unavailable`; do not render unsafe protocols.
- Hover/focus style: blue underline.

### Confidence Badge

- Visible label: confidence enum.
- Shape: rounded pill.
- No icons.
- Tooltip/title: human-readable confidence explanation.
- Colors: use semantic badge colors from the Color section, not accent blue.

### Loading, Empty, Error, And Stale States

- Loading: skeleton rows that mimic table cell structure; do not use a spinner-only loading state.
- No data: centered neutral empty state with the empty copy above.
- No matches after filtering: keep controls visible and show the filter-empty copy above in the table body area.
- Load error: warning banner above the table controls with the error copy above and a retry action.
- Retry behavior: use full page refresh unless a client-side refetch path already exists.
- Stale data: if latest collection is older than 24 hours, show an amber warning banner above the table.
- Freshness metadata: show both last collection time and next scheduled run time when available.

### Pagination

- Page size: 50 rows.
- Controls: Previous and Next buttons plus page indicator.
- Disabled state: opacity 50% and disabled cursor.
- Pagination must reset to page 1 when filters reduce the result set.

## Data And Formatting Contract

- Server component fetches pricing rows with a join from `extractions` to `sources`.
- The table receives plain serializable props; client interactivity stays in `PricingTable`.
- Deduplicate to the latest extraction per model and provider before passing rows to the client.
- When different providers report prices for the same model, show both provider rows.
- Use `formatCurrencyPrice()` for all price rendering.
- Use `formatContextWindow()` for context window values.
- Use `sanitizeDisplayName()` for all user-visible model and provider names.
- Use `getModelFamily()` for family display and family search.
- Use `generateSlug(modelName, sourceId)` for model detail links.
- Null prices display `N/A` plus the null price tooltip.
- Last-updated should derive from max `extractions.collectedAt`, not source row update time.

Source: context D-21 through D-28, code insights, and existing utility modules.
