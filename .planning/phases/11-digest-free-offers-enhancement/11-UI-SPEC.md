## Interaction Patterns

### Hover States
- Card background opacity increases: `bg-green-500/10` -> `bg-green-500/15` (free), `bg-amber-500/10` -> `bg-amber-500/15` (promo)
- Provider link: `text-accent-blue` -> `text-accent-blue-hover`
- Transition: `transition-colors` (default Tailwind duration)

### External Links
- All provider links open in new tab: `target="_blank" rel="noopener noreferrer"`
- External link icon: inline SVG (16x16, `w-3.5 h-3.5`)
- Icon path: standard "open in new window" arrow

### Loading States
- 3 skeleton cards with `animate-pulse`
- Skeleton bars match card structure: badge placeholder, title placeholder, description placeholder, link placeholder
- Skeleton colors use 5% / 20% opacity of the section's semantic color

### Error States
- Section heading still visible
- Error message in `text-text-tertiary text-sm`
- No retry button (data refreshes on page reload via ISR)

## Theme Compatibility

All Phase 11 components must work in both light and dark themes. The existing implementations already use:

- CSS custom properties for all colors (via `globals.css`)
- `dark:` variant overrides where Tailwind opacity utilities don't auto-adapt
- Semantic token classes: `text-text-primary`, `text-text-secondary`, `text-text-tertiary`, `text-accent-blue`, etc.

**No hardcoded hex colors in component JSX.** All color comes from either:
1. Tailwind opacity utilities on semantic colors (e.g., `bg-green-500/10`)
2. CSS variable-backed token classes (e.g., `text-text-primary`)
3. `dark:` variant overrides for cases where opacity doesn't adapt (e.g., `text-green-600 dark:text-green-400`)

## Data Flow Contract

### API Endpoint
- **Route:** `/api/digest-promotions?date={YYYY-MM-DD}`
- **Response shape:**
  ```typescript
  {
    freeOffers: Array<{
      id: number;
      modelPattern: string;
      type: string;
      description: string;
      credits: string | null;
      sourceName: string;
      sourceUrl: string;
    }>;
    promotions: Array<{
      id: number;
      modelPattern: string;
      type: string;
      description: string;
      credits: string | null;
      sourceName: string;
      sourceUrl: string;
    }>;
  }
  ```
- **Source:** Queries `promotions` table joined with `sources` table
- **Filter:** By date, split by `type` (free_tier vs promotion)

### Archive Page Badges
- **Current:** Regex-based extraction from article markdown content
- **Phase 11 improvement:** Could query promotions table per date for accurate counts
- **Decision needed:** Keep regex extraction (fast, no extra queries) or switch to DB queries (accurate but N+1)
