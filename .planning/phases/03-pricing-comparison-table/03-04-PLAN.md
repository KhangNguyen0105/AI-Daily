  <action>
    Update app/lib/pricing-utils.ts to add currency conversion and VND formatting:

    1. **USD_VND_RATE constant**: Export `const USD_VND_RATE = 25500` as the hardcoded exchange rate (1 USD = 25,500 VND).

    2. **convertToVND function**: Export a function `convertToVND(price: number | null | undefined): number | null | undefined` that multiplies the price by USD_VND_RATE. Returns null/undefined as-is (passthrough for missing data).

    3. **formatVND function**: Export a function `formatVND(priceInVND: number | null | undefined): string` that:
       - Returns "N/A" for null/undefined
       - Returns "0 ₫" for 0
       - For positive values: rounds to nearest integer, formats with dot as thousands separator (Vietnamese convention), appends " ₫"
       - Uses `toLocaleString('vi-VN')` for number formatting, then appends " ₫"
       - Example: 89250 -> "89.250 ₫", 1000000 -> "1.000.000 ₫"

    4. **formatCurrencyPrice function**: Export a function `formatCurrencyPrice(price: number | null | undefined, currency: 'usd' | 'vnd'): string` that:
       - If currency is 'usd': delegates to existing formatPrice(price)
       - If currency is 'vnd': calls convertToVND(price) then formatVND()
       - This is the single entry point that PricingTable will use

    Update tests/pricing-utils.test.ts to add test suites for:
    - convertToVND (including null/undefined passthrough)
    - formatVND (including Vietnamese number formatting)
    - formatCurrencyPrice (both USD and VND modes, including edge cases)

    Existing formatPrice tests must remain unchanged and passing.
