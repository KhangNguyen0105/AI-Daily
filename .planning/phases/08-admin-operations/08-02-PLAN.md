<objective>
Build the admin dashboard layout with sidebar navigation, header bar, and overview page showing pipeline health summary cards. This establishes the visual framework that all subsequent admin pages (articles, pipeline, sources) will render within. Per D-04, D-05, D-06.

Purpose: The admin layout is the container for all admin functionality. Without it, there is no navigation between admin sections and no consistent header/footer. The overview page provides the landing page after login with at-a-glance system health.

Output: Working admin layout with sidebar, header, and overview dashboard at /admin.
</objective>
