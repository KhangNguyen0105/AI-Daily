# Phase 8 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (plain Tailwind utilities + TanStack Table) |
| Icon library | none (SVG inline or emoji) |
| Font | system-ui (Tailwind v4 default) |

**Rationale:** Project uses Tailwind CSS 4.3.0 with `@tailwindcss/postcss`. No `components.json` exists. No shadcn. `globals.css` contains only `@import "tailwindcss"` -- no custom `@theme` tokens. All styling uses Tailwind v4 defaults. TanStack Table (already used in PricingTable.tsx) provides the table primitive. CONTEXT.md (D-15 through D-30) locks all layout and component decisions.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, table cell padding |
| sm | 8px | Compact element spacing, sidebar link padding |
| md | 16px | Default element spacing, card padding, page horizontal padding |
| lg | 24px | Section padding, sidebar width internal padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks, admin layout top offset |
| 3xl | 64px | Not used in this phase |

Exceptions: 44px minimum touch target for toggle switches and mobile hamburger button (accessibility requirement).

**Source:** Derived from existing codebase patterns. `px-4` (16px) used in TopNav, PricingTable, HomePageClient. `pb-16` (64px) for section breaks. `gap-6` (24px) for nav items. `py-3` (12px) for nav padding. These are the established spacings.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 12px (text-xs) | 600 | 1.4 |
| Body | 14px (text-sm) | 400 | 1.5 |
| Heading | 18px (text-lg) | 600 | 1.3 |
| Display | 24px (text-2xl) | 700 | 1.3 |

**Font families:** Default system-ui stack (Tailwind v4 default). No custom fonts.

**Source:** TopNav uses `text-lg font-bold` for brand, `text-sm` for links, `text-sm font-bold` for active links. HomePageClient uses `text-2xl font-semibold` for section headings. PricingTable uses `text-sm` for cell content. These are the established patterns.

**Admin-specific additions:**
- Sidebar nav link: 14px (text-sm), weight 400, active state weight 600
- Summary card value: 24px (text-2xl), weight 700
- Summary card label: 14px (text-sm), weight 400, color gray-500
- Table header: 12px (text-xs), weight 600, uppercase, letter-spacing wide
- Table cell: 14px (text-sm), weight 400
- Form label: 14px (text-sm), weight 500
- Form input: 14px (text-sm), weight 400
- Error message: 14px (text-sm), weight 400, color red-600

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #ffffff (white) | Page background, card surfaces, sidebar background |
| Secondary (30%) | #f9fafb (gray-50) | Table row alternating, sidebar hover states, section backgrounds |
| Accent (10%) | #2563eb (blue-600) | Active nav links, primary buttons, toggle switch active state |
| Destructive | #dc2626 (red-600) | Rollback confirmation, delete actions, error text |

**Accent reserved for:**
- Active sidebar navigation link text
- Primary action buttons ("Save", "Re-crawl", "Regenerate", "Publish")
- Toggle switch active/checked state
- Links within admin content

**Never used accent for:** Borders, backgrounds, decorative elements, inactive states.

**Additional semantic colors (from existing codebase):**
- Success: #16a34a (green-600) -- successful pipeline run, published status badge
- Warning: #ca8a04 (yellow-600) -- low confidence badge, auto-publish disabled state
- Error: #dc2626 (red-600) -- failed pipeline run, error log entries, destructive confirmations
- Muted: #6b7280 (gray-500) -- secondary text, timestamps, placeholder text

**Source:** TopNav uses `bg-white`, `border-gray-200`, `text-gray-900`, `text-gray-600`, `text-blue-600` (active). HomePageClient uses `text-gray-500` for subtitles. These are the established tokens. No custom `@theme` overrides exist in `globals.css`.

## Interactive Behaviors

### Sidebar Mobile Collapse
- Below `md` breakpoint (768px): sidebar hidden by default
- Hamburger icon in AdminHeader toggles sidebar overlay
- Overlay: sidebar slides in from left, semi-transparent backdrop behind it
- Click backdrop or nav link closes overlay
- Transition: transform translate-x, 200ms ease

### Table Expandable Rows
- Click row or chevron icon to expand/collapse detail section
- Only one row expanded at a time (collapse previous when new one opens)
- Expand animation: max-height transition, 150ms ease
- Chevron rotates 180 degrees when expanded

### Toggle Switches (Trust, Auto-publish)
- Immediate save on toggle (no separate save button)
- Show loading spinner briefly during save (optimistic UI)
- Toast notification on success or failure
- On failure: revert toggle to previous state

### Confirmation Dialogs
- Modal overlay (fixed, bg-black/50)
- Centered card (max-w-md, white bg, rounded-lg, p-6)
- Two buttons: "Cancel" (gray outline) and action-specific confirm button:
  - Rollback: "Yes, Rollback" (red bg)
  - Re-crawl: "Yes, Re-crawl" (red bg)
  - Regenerate: "Yes, Regenerate" (red bg)
- Press Escape or click backdrop to dismiss
- Focus trapped inside modal when open

### Toast Notifications
- Fixed position: bottom-right (bottom-6 right-6)
- Auto-dismiss: 5 seconds
- Manual dismiss: click X button
- Max 3 visible at once (queue additional)
- Success: green left border, white bg
- Error: red left border, white bg
- Slide-in animation from right, 200ms

### Article Edit: Tab Switching
- "Edit" and "Preview" tabs above textarea
- Active tab: blue-600 bottom border, font-semibold
- Inactive tab: gray-500 text, no border
- Preview renders Markdown with basic HTML (h1-h3, p, ul/ol, code, pre, a, strong, em)
- No full Markdown library needed -- use a lightweight renderer or simple regex

### Loading States
- Page-level: skeleton placeholders matching layout structure (gray-200 rounded rectangles)
- Table-level: "Loading..." text in table body with spinner
- Button-level: spinner icon + disabled state during async actions
- Summary cards: pulse animation on skeleton while loading

## Accessibility

- All interactive elements have visible focus rings (focus-visible:ring-2 focus-visible:ring-blue-600)
- Toggle switches use `<button>` with `role="switch"` and `aria-checked`
- Tables use `<thead>`, `<th scope="col">`, proper `<tbody>` structure
- Confirmation dialog traps focus, returns focus to trigger on close
- Error messages linked to inputs via `aria-describedby`
- Sidebar navigation uses `<nav aria-label="Admin navigation">`
- Skip-to-content link not needed (admin-only, short pages)
- Color is never the sole indicator (always paired with icon or text)
