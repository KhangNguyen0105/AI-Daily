Create `src/pipeline/article-generator.ts` with the following:

Export a `GeneratedArticle` interface with fields: `title` (string), `summary` (string), `content` (string — Markdown).

Export an async `generateArticle(diff: DiffResult)` function that:

1. Reads `env.AI_PROVIDER` and `env.AI_FALLBACK_PROVIDER` from `@/src/lib/env.ts`.
2. Defines a `getModel(provider: string)` helper that returns the appropriate model object:
   - `'anthropic'` -> `anthropic('claude-sonnet-4-5')` from `@ai-sdk/anthropic`
   - `'openai'` -> `openai('gpt-4o')` from `@ai-sdk/openai`
   - `'mimo'` -> `openai('mimo-v2.5-pro', { baseURL: env.MIMO_BASE_URL, apiKey: env.MIMO_API_KEY })` from `@ai-sdk/openai` (MIMO uses OpenAI-compatible API)
3. Builds a context string from the DiffResult summarizing the changes (compact format, not raw JSON). Note on D-06: confidence scores are not included in the DiffResult — they are already embedded in the extractions table and would add significant prompt size. The compact diff format focuses on what changed (model names, prices, promotions) which is what readers care about. Confidence scores are visible on the pricing table (Phase 3) and model detail pages (Phase 5).
4. Constructs a system prompt that instructs the LLM to:
   - Write a daily AI pricing digest in exactly 4 sections: `## Key Changes`, `## Pricing Highlights`, `## What to Watch`, preceded by a title line as `# {title}`
   - Use neutral, factual tone (per D-04): "GPT-4o input price dropped 20% to $2.50/1M tokens"
   - Keep it 300-500 words (per D-03)
   - Return ONLY the article content, no preamble or wrapping
   - When no changes detected (empty diff), write a "no changes" article per D-05: "No pricing changes detected today. {totalModelsToday} models tracked."
   - Include a one-line summary (max 150 chars) on the very first line, preceded by `SUMMARY: ` delimiter
   - Include the article title on the second line as `# {title}`
5. Constructs a user prompt with the diff context string.
6. Calls `generateText()` from `ai` package with `temperature: 0.3` and `maxOutputTokens: 1024`.
7. Wraps in try/catch — if primary provider fails, retries with fallback provider.
8. Parses the response text to extract:
   - `summary`: text after `SUMMARY: ` delimiter on the first line
   - `title`: text after `# ` on the second line
   - `content`: everything after the title line (the actual Markdown article)
   - If parsing fails (delimiters not found), use fallback: title = "AI Daily Digest", summary = first 150 chars of response, content = full response text.

Create `tests/pipeline/article-generator.test.ts` with tests covering:
- generateArticle calls generateText with the primary provider model
- generateArticle falls back to secondary provider when primary throws
- Response parsing extracts title, summary, and content from delimited output
- Response parsing handles missing delimiters gracefully (fallback extraction)
- Empty diff input produces a "no changes" article in the prompt context
- Provider model selection returns correct model for 'anthropic', 'openai', 'mimo'

Note: Tests should mock the `generateText` function from `ai` package and the env module. Use `vi.mock('ai')` and `vi.mock('@/src/lib/env')`.
