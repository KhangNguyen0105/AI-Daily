  - If no articles: show empty state message "No articles yet. Check back after the first pipeline run."
  - If articles exist: render a list where each item shows:
    - Date formatted as "MMMM d, yyyy" (e.g., "June 14, 2026") using date-fns `format`
    - Title as a link to `/digest/{date}`
    - Summary text below the title
  - A "Load more" button at the bottom. Since this is a server component, implement "Load more" as a simple link to `/digest?offset=30` that triggers a re-query with `.offset(30)`. Step 3: Create `app/digest/[date]/page.tsx` — the article detail page.

Follow the exact pattern from `app/model/[slug]/page.tsx`:
- Export `revalidate = 60`.
- Export `generateStaticParams` that queries the 90 most recent article dates: `db.select({ date: articles.date }).from(articles).orderBy(desc(articles.date)).limit(90)`. Return `{ date: row.date }` for each. Wrap in try/catch returning empty array.
- The default export is an async server component that:
  - Awaits `params: Promise<{ date: string }>`.
  - Validates date format with regex `/^\d{4}-\d{2}-\d{2}$/`. If invalid, call `notFound()`.
  - Queries: `db.select().from(articles).where(eq(articles.date, date)).limit(1)`.
  - If no article found, call `notFound()`.
  - Wraps in try/catch with `notFound()` fallback.
  - Renders the article using the `DigestArticle` component (created in Task 2): `<DigestArticle article={article} />`.

Use `import { notFound } from 'next/navigation'` and `import { eq, desc } from 'drizzle-orm'`.
