<objective>
Build the pipeline action triggers (re-crawl, regenerate, run-full, cancel) and auto-publish toggle, along with their supporting API routes. These components integrate with the existing BullMQ queues to give the admin direct control over the automated pipeline. Per D-01, D-02, D-03, D-04, D-06.

Purpose: These are the operational controls that let the admin manage data quality. The triggers queue jobs to existing BullMQ workers. The auto-publish toggle controls whether low-confidence items are published automatically. All destructive actions require confirmation.

Output: Working re-crawl, regenerate, run-full, cancel triggers and auto-publish toggle on the pipeline page.
</objective>

  <action>
  1. Create `app/api/admin/pipeline/re-crawl/route.ts` (per Pattern 4 from PATTERNS.md):
     - Import db, sources, eq from drizzle-orm
     - Import collectQueue from @/src/pipeline/queues
     - Import auth, z, NextResponse
     - Auth check at top
     - POST handler:
       - Validate: z.object({ providerName: z.string().min(1) })
       - Verify provider exists: db.select().from(sources).where(eq(sources.name, providerName)).limit(1)
       - If not found, return 404
       - Enqueue: collectQueue.add('collect', { providerName, pipelineRunId: null }, { jobId: `re-crawl-${providerName}-${Date.now()}` })
       - Return { success: true, message: `Re-crawl job queued for ${providerName}` }

  2. Create `app/api/admin/pipeline/regenerate/route.ts`:
     - Import generateQueue from @/src/pipeline/queues
     - Import auth, z, NextResponse
     - Auth check at top
     - POST handler:
       - Validate: z.object({ date: z.string().regex(/^\d{4}-\d{2}-\d{2}$/) })
       - Enqueue: generateQueue.add('generate', { date, force: true }, { jobId: `regenerate-${date}-${Date.now()}` })
       - Return { success: true, message: `Article regeneration queued for ${date}` }

  3. Create `app/api/admin/pipeline/run-full/route.ts`:
     - Import orchestrateDailyRun from @/src/pipeline/orchestrator
     - Import auth, NextResponse
     - Auth check at top
     - POST handler:
       - Call orchestrateDailyRun() in background (don't await — let it run)
       - Return { success: true, message: "Full pipeline run started" }

  4. Create `app/api/admin/pipeline/cancel/route.ts`:
     - Import pipelineRuns, eq from drizzle-orm
     - Import auth, z, NextResponse
     - Auth check at top
     - POST handler:
       - Validate: z.object({ runId: z.number() })
       - Update pipeline run status to 'cancelled'
       - Return { success: true }

  5. Create `app/api/admin/settings/route.ts` (per Pattern 15 from PATTERNS.md):
     - Import db, adminSettings from @/src/db/schema
     - Import auth, z, NextResponse
     - Auth check at top
     - GET handler: query all settings, return as key-value object. Default: { auto_publish: "true" }
     - PUT handler:
       - Validate: z.object({ key: z.string().min(1), value: z.string() })
       - Upsert: db.insert(adminSettings).values({ key, value }).onConflictDoUpdate({ target: adminSettings.key, set: { value, updatedAt: new Date() } })
       - Return { success: true }
  </action>
