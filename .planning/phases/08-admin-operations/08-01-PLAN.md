## Phase Goal

**As an** admin, **I want to** monitor the pipeline, manage published content, and control data quality through a secure dashboard, **so that** I can ensure the automated system produces accurate, timely AI pricing intelligence.

<objective>
Set up the authentication layer for the admin dashboard: install NextAuth.js v5, configure Credentials provider with JWT sessions, create middleware to protect all /admin/* routes and /api/admin/* API routes, build a custom login page, and add the articleVersions and adminSettings tables to the Drizzle schema. Per D-01, D-02, D-07, D-08, D-09, D-10.

Purpose: Authentication is the foundation for all admin functionality. Without it, no admin page or API route can be accessed. The schema extensions are included here because they have no dependencies on other plans and must exist before the articles and pipeline plans can use them.

Output: Working login flow at /admin/login, protected admin routes, session provider for admin layout, two new database tables in schema.
</objective>

     - Import `auth` from `@/src/auth`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client->login | Password sent from browser to NextAuth API route |
| session->middleware | JWT token validated on every /admin/* request |
| middleware->api | /api/admin/* routes protected by middleware auth check |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing | Login form | mitigate | NextAuth Credentials provider validates password against ADMIN_PASSWORD env var. JWT signed with NEXTAUTH_SECRET. |
| T-08-02 | Tampering | JWT session | mitigate | NextAuth JWT is signed with NEXTAUTH_SECRET — tampering invalidates the token. 8-hour expiry limits window. |
| T-08-03 | Information Disclosure | ADMIN_PASSWORD | mitigate | Password stored in env var, never exposed to client. NEXTAUTH_SECRET required for JWT signing. |
| T-08-04 | Denial of Service | Login endpoint | accept | Single admin user, no rate limiting needed. Strong password is the mitigation. |
| T-08-SC | Tampering | npm install (next-auth) | mitigate | Verify next-auth package on npmjs.com before install. Package is well-known (1M+ weekly downloads). |
</threat_model>
